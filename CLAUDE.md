# TODO アプリ

## アーキテクチャ

アプリケーションは Vite + React + TypeScript + TailwindCSS で作成される。

## 要件

### 概要

アプリケーションは TODO を表示・作成・編集・削除できる。

## TODO の永続化

アプリケーションのビルド時に TODO の保存方法を以下の中から選択できるようにする。

- オンメモリ (単体テスト、E2E テスト用)
- Local Storage (本番用)

### TODO

TODO は以下の情報を持つ。

- ID
- 題名
- 説明
- 状態 (新規、実施中、完了、不要)
- 担当者
- 期限

### 画面の一覧

アプリケーションは以下の画面を持つ。

- TODO 一覧表示画面
- TODO 作成画面
- TODO 編集画面
- TODO 削除確認画面

## 単体テスト

`*.ts`, `*.tsx`のファイルについては Vitest を用いた単体テストを行う。

テスト観点は以下とする。

| 分類       | テスト観点                                                                   |
| ---------- | ---------------------------------------------------------------------------- |
| 画面の部品 | 初期表示と、画面上の各部品の操作を行ったとき、期待動作を満たすことを確認する |
| 画面       | 初期表示と、画面上の各部品の操作を行ったとき、期待動作を満たすことを確認する |
| ロジック   | 分岐網羅を満たすことを確認する                                               |

## E2E テスト

Playwright を用いた E2E テストを行う。

テスト内容は以下とする。

1. `/`にアクセスしたとき、TODO 一覧表示画面が表示されること。
1. `新規TODO作成`ボタンを押したとき、TODO 作成画面が表示されること。
1. TODO 作成画面にある`キャンセル`ボタンを押したとき、TODO 一覧表示画面に戻ること。
1. TODO 作成画面で全ての項目を入力し`作成`ボタンを押したとき、TODO が登録された TODO 一覧表示画面が表示されること。
1. TODO 一覧表示画面にある TODO の`編集`ボタンを押したとき、TODO 編集画面が表示されること。
1. TODO 編集画面にある`キャンセル`ボタンを押したとき、TODO 一覧表示画面に戻ること。
1. TODO 編集画面で全ての項目を入力し`更新`ボタンを押したとき、TODO が登録された TODO 一覧表示画面が表示されること。
1. TODO 一覧表示画面にある TODO の`削除`ボタンを押したとき、TODO 削除確認画面が表示されること。
1. TODO 削除確認画面にある`キャンセル`ボタンを押したとき、TODO 一覧表示画面に戻ること。
1. TODO 削除確認画面にある`削除`ボタンを押したとき、TODO が削除された TODO 一覧表示画面が表示されること。
